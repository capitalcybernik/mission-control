## During-Event Roles & Responsibilities

### Moderator (Rick Dassler or Nikhil)

**Primary Responsibility:** Keep the event running on schedule and facilitate a smooth, professional experience for all attendees and speakers.

**Key Tasks:**
- Start the event exactly at 12:00 PM with a warm welcome and brief opening remarks (1-2 minutes)
- Introduce each speaker with their name, title, company, and session topic
- Monitor the time closely and provide speakers with a 2-minute warning before their 15 minutes ends
- Facilitate smooth transitions between speakers
- Manage the consolidated Q&A session at the end (1:00 PM - 1:15 PM)
- Select the most relevant and interesting questions from the Q&A feature
- Provide closing remarks and thank all attendees and speakers
- Ensure the event ends on time at 1:30 PM

### Tech Support Lead

**Primary Responsibility:** Ensure all technical aspects of the event run smoothly and troubleshoot any issues that arise.

**Key Tasks:**
- Admit attendees from the waiting room starting at 11:55 AM
- Monitor speaker connections and audio/video quality
- Assist any speaker experiencing technical difficulties
- Manage screen sharing transitions between speakers
- Monitor the chat for technical questions from attendees
- Start recording at 12:00 PM and ensure continuous recording throughout
- Monitor for any platform issues or glitches
- Have backup connection information ready for all speakers

### Chat Monitor

**Primary Responsibility:** Monitor Q&A submissions and manage attendee engagement.

**Key Tasks:**
- Monitor the Q&A feature throughout all presentations
- Flag relevant questions for the Moderator to ask during Q&A session
- Remove any inappropriate or off-topic questions
- Respond to technical questions from attendees in real-time
- Track engagement metrics (number of questions, participation rate)
- Prepare summary of top questions for post-event analysis

### Timekeeper

**Primary Responsibility:** Track speaker times and alert the Moderator of timing.

**Key Tasks:**
- Track each speaker's 15-minute presentation time
- Provide visual signals to the Moderator (e.g., 2-minute warning, time's up)
- Track the consolidated Q&A session timing
- Ensure event ends by 1:30 PM
- Document actual times for post-event analysis

## Q&A Management Workflow

**At Event Start (12:00 PM):**
- Moderator instructs attendees to submit questions via the Q&A feature
- Explain that questions will be answered in a consolidated Q&A session at the end
- Encourage attendees to ask questions throughout the presentations

**During Presentations (12:00 PM - 1:00 PM):**
- Chat Monitor continuously monitors Q&A submissions
- Chat Monitor flags the most relevant and interesting questions
- Chat Monitor removes any inappropriate or off-topic questions
- Prepare 8-10 top questions for the Q&A session

**During Q&A Session (1:00 PM - 1:15 PM):**
- Moderator asks selected questions to the appropriate speaker
- Timekeeper ensures Q&A stays within 15-minute window
- Prioritize questions that benefit the broadest audience
- If time permits, take 1-2 questions from chat during Q&A

## Contingency Planning

### Speaker No-Show
- Have backup speaker contact information ready
- Prepare brief remarks to fill the time if needed
- Adjust agenda if necessary

### Technical Issues
- Have backup internet connection for Moderator and Tech Support
- Have backup Zoom link ready
- Test all backup systems before event

### Low Attendance
- Proceed with event as planned
- Focus on quality engagement with attendees present
- Use as opportunity to gather detailed feedback

### Platform Issues
- Have alternative platform ready (e.g., Teams, Google Meet)
- Test backup platform before event day
- Know how to quickly migrate to backup platform if needed

## Post-Event Analysis

**Metrics to Track:**
- Total registrations and actual attendance
- Attendee engagement (questions asked, chat participation)
- Speaker performance (feedback, engagement)
- Technical issues encountered and resolution time
- Attendee satisfaction rating
- Lead generation and follow-up conversion

**Feedback Collection:**
- Send post-event survey to attendees (within 24 hours)
- Collect feedback from speakers (within 48 hours)
- Gather feedback from event team (during debrief meeting)

**Success Criteria:**
- 80%+ attendance rate
- 4.0+ average satisfaction rating
- 50+ qualified leads generated
- 0 major technical issues
- All speakers complete on time
